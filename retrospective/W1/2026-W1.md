## 2. 회고 (KPT — 과정 개선)

### Keep (계속 유지할 행동)
- 미션이 요구하는 논리적 순서를 그대로 따라가며 분석을 진행한 점
- 에러를 그냥 넘기지 않고 원인(문법 차이, 라이브러리 표기 차이 등)까지 파고들어 확인한 점
- LLM에 의존하지 않고 직접 검색하며 코드를 작성한 점
- 제출한 코드를 다시 처음부터 점검하는 습관

### Problem (문제가 발생한 행동)
- 시각화나 컬럼 매핑 등에서 "일단 동작하면 넘어가는" 식으로 세부 검증을 미룬 경우가 반복됨 (x축 라벨, Region 컬럼 인덱스)
- 새 도구/라이브러리를 쓸 때 사전 조사 없이 기존 지식(MySQL 문법, 특정 라이브러리 국가명 표기)을 그대로 적용하려다 실패함
- 코드 리뷰 타이밍이 하루 늦어져 문제 발견이 지연됨

### Try (다음 주에 새롭게 시도할 행동)
- 시각화/매핑 코드 작성 시 "결과가 실제로 의미하는 라벨과 일치하는지"를 완료 기준에 포함시키기
- 새로운 라이브러리·DB 도입 시 공식 문서에서 지원 문법/호환성을 먼저 5분이라도 확인하고 시작하기
- 코드 작성 당일 1차 리뷰 습관을 이번 주부터 실제로 적용해보고, 그 효과를 다음 주 회고에서 점검하기
