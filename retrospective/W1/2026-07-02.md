## 2. 회고 (KPT — 과정 개선)

#### Keep (잘해서 계속 유지할 행동)
- 에러 메시지를 그대로 넘기지 않고 원인(DB 문법 차이)까지 확인하고 넘어간 것
- 학습용 DB를 처음부터 직접 만들어보며 스키마 구조를 손으로 익힌 것


#### Problem (문제가 발생한 행동)
- SQL 문법이 DB마다 다르다는 걸 간과하고 MySQL/표준 SQL 스타일의 TOP 구문을 SQLite에 그대로 쓰려 했음
- 파일 경로를 상대경로 서브디렉토리로 바꿀 때, 상위 디렉토리 생성 여부를 미리 고려하지 않고 실행 후에야 에러로 발견함


#### Try (다음에 새롭게 시도할 행동)
- 새로운 DB/라이브러리를 쓸 때는 코드를 짜기 전에 지원 문법(LIMIT vs TOP, 윈도우 함수 지원 여부 등)을 먼저 확인하기
- 파일 입출력 코드를 작성할 때 처음부터 os.makedirs(exist_ok=True) 패턴을 습관적으로 넣기
