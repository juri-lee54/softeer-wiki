# 워드클라우드(Word Cloud)의 내부 동작 원리

### 워드클라우드(Word Cloud)란?

워드클라우드는 텍스트 데이터에서 **단어의 등장 빈도**를 분석하여, 많이 등장한 단어일수록 더 크게 표시하는 시각화 기법이다.

단순히 단어를 크게 출력하는 것이 아니라, 텍스트를 분석하고 단어의 중요도를 계산한 뒤, 크기·위치·색상을 결정하여 하나의 이미지로 생성하는 과정을 거친다.

`WordCloud.generate(text)`를 호출하면 내부적으로 다음과 같은 순서로 동작한다.

<br>

## 1. 텍스트 전처리 및 단어 추출

가장 먼저 입력된 텍스트를 분석하여 단어를 추출한다.

### 1) 토큰화(Tokenization)

긴 문자열을 단어(Token) 단위로 분리한다.

예를 들어

```text
I really love this game.
```

는

```text
["I", "really", "love", "this", "game"]
```

처럼 단어 목록으로 변환된다.

WordCloud는 기본적으로 정규식(`\w[\w']+`)을 이용하여 단어를 추출하며, 숫자로만 이루어진 토큰이나 너무 짧은 단어는 제외될 수 있다.

---

### 2) 불용어(Stopwords) 제거

불용어는 자주 등장하지만 의미 전달에 큰 영향을 주지 않는 단어를 의미한다.

예를 들어

* the
* is
* a
* of
* to

등이 이에 해당한다.

`stopwords` 옵션을 지정하면 이러한 단어들은 빈도 계산에서 제외된다.

이번 과제에서는 `clean_text()` 함수에서 이미 `ENGLISH_STOP_WORDS`를 이용하여 불용어를 제거했기 때문에 WordCloud 내부의 불용어 제거 과정은 큰 영향을 주지 않는다.

---

### 3) 연어(Collocation) 탐지

단어를 개별적으로만 분석하는 것이 아니라, **함께 자주 등장하는 단어 조합**도 하나의 단위로 처리할 수 있다.

예를 들어

* machine learning
* artificial intelligence
* customer service

와 같은 표현은 각각 하나의 의미를 가지므로, 따로 분리하는 것보다 하나의 단어처럼 처리하는 것이 더 적절하다.

`collocations=True`(기본값)를 사용하면 WordCloud는 두 단어가 함께 등장하는 빈도를 분석하고, **Dunning Log-Likelihood**라는 통계적 방법을 이용하여 우연이 아니라 실제로 자주 함께 사용되는 표현인지를 판단한다.

의미 있는 조합이라고 판단되면

```text
machine learning
```

을

```text
machine_learning
```

과 같은 하나의 토큰으로 처리하여 이후 과정에서 하나의 단어처럼 다룬다.

---

### 4) 단어 빈도 계산

전처리가 끝나면 단어별 등장 횟수를 계산한다.

예를 들어

```text
love game love fun game love
```

이라면

| 단어   | 등장 횟수 |
| ---- | ----: |
| love |     3 |
| game |     2 |
| fun  |     1 |

처럼 단어별 빈도를 집계한다.

내부적으로는 Python의 `collections.Counter`와 유사한 방식으로 빈도를 계산한다.

---

### 5) 상위 단어 선택

모든 단어를 시각화하면 너무 복잡해지기 때문에 등장 횟수가 높은 단어만 선택한다.

예를 들어

```python
max_words=200
```

이라면 빈도가 높은 상위 200개의 단어만 사용하고, 나머지는 제외한다.

<br>

## 2. 단어 크기(Font Size) 계산

단어의 크기는 등장 빈도를 기반으로 계산된다.

가장 많이 등장한 단어를 기준으로 삼고, 다른 단어들은 그에 대한 **상대적인 비율**로 크기가 결정된다.

이때 `relative_scaling` 옵션이 크기 계산 방식에 영향을 준다.

* `relative_scaling = 0`

  * 실제 빈도보다 **순위**를 기준으로 크기를 결정한다.

* `relative_scaling = 1`

  * 실제 등장 빈도 비율을 그대로 반영한다.

* 기본값(`auto`)

  * 일반적으로 `generate()`에서는 약 **0.5**로 동작한다.
  * 즉, 순위와 실제 빈도를 적절히 혼합하여 크기를 결정한다.

그래서 가장 많이 등장한 단어는 충분히 크게 강조하면서도, 전체적인 크기 차이는 너무 극단적이지 않도록 조절된다.

<br>

## 3. 단어 배치(Layout)

단어 크기가 결정되면 화면에 단어를 배치한다.

WordCloud는 **큰 단어부터 작은 단어 순서**로 배치를 진행한다.

### 1) 초기 위치 선정

각 단어는 먼저 캔버스 중앙 근처의 임의 위치에서 배치를 시도한다.

---

### 2) 겹침 검사

배치하려는 위치에 이미 다른 단어가 있는지 검사한다.

이때 WordCloud는 **적분 이미지(Integral Image)** 기법을 이용한다.

적분 이미지는 특정 영역이 이미 사용 중인지 매우 빠르게 확인할 수 있는 자료구조로, 단어가 많아져도 충돌 검사를 효율적으로 수행할 수 있다.

---

### 3) 나선형 탐색

만약 다른 단어와 겹친다면 해당 위치에는 배치할 수 없다.

이 경우 **아르키메데스 나선(Archimedean Spiral)** 경로를 따라 조금씩 이동하면서 빈 공간을 탐색한다.
![나선](https://upload.wikimedia.org/wikipedia/commons/1/1e/%EC%95%84%EB%A5%B4%ED%82%A4%EB%A9%94%EB%8D%B0%EC%8A%A4_%EB%82%98%EC%84%A0.png)

빈 공간을 찾으면 그 위치에 단어를 배치하고, 다음 단어도 같은 과정을 반복한다.

이러한 방식 덕분에 단어들이 서로 겹치지 않으면서도 화면을 빈틈없이 채울 수 있다.

---

### 4) 단어 방향 결정

기본적으로는 대부분의 단어를 가로 방향으로 배치한다.

`prefer_horizontal=0.9`가 기본값이므로

* 약 90%는 가로 방향
* 약 10%는 세로 방향(90° 회전)

으로 배치된다.

워드클라우드에서 일부 단어가 세로로 보이는 이유가 바로 이 설정 때문이다.

---

### 5) 마스크(Mask) 적용

기본적으로는 사각형 캔버스 안에 단어를 배치한다.

하지만 `mask` 이미지를 지정하면

* 하트
* 별
* 회사 로고
* 사람 얼굴

등 원하는 모양 안에서만 단어가 배치된다.

---

### 6) 무작위성(Randomness)

단어의 시작 위치와 탐색 과정에는 일부 무작위 요소가 포함되어 있다.

따라서 같은 텍스트라도 실행할 때마다 단어의 위치가 조금씩 달라질 수 있다.

동일한 결과를 얻고 싶다면

```python
random_state=42
```

처럼 난수 시드를 고정하면 항상 같은 형태의 워드클라우드를 생성할 수 있다.

<br>

## 4. 색상(Color) 지정

단어의 위치가 결정되면 색상을 지정한다.

기본적으로는 `colormap`에서 색상을 무작위로 선택한다.

즉,

* 단어의 크기와 색상은 서로 관련이 없으며,
* 색상은 시각적인 구분을 위한 요소이다.

필요하다면 `color_func`를 이용하여

* 긍정 단어는 파란색
* 부정 단어는 빨간색

처럼 원하는 기준으로 직접 색상을 지정할 수도 있다.

<br>

## 5. 이미지(Rendering) 생성

모든 단어의 크기, 위치, 색상이 결정되면 WordCloud는 이를 하나의 **RGB 이미지**로 렌더링한다.

내부적으로는 이미지를 **NumPy 배열** 형태로 저장하며, `to_array()`를 이용하면 실제 픽셀 데이터를 확인할 수 있다.

워드클라우드는 텍스트 객체가 아니라 이미지 객체이므로

```python
plt.imshow(wordcloud, interpolation="bilinear")
```

처럼 `imshow()`를 이용하여 출력한다.

여기서

```python
interpolation="bilinear"
```

는 인접한 픽셀을 부드럽게 보간하는 옵션이다.

이 옵션을 사용하면 글자의 곡선 부분이 계단처럼 보이지 않고, 확대하거나 축소해도 보다 자연스럽고 선명하게 표현된다.

<br>

## 전체 동작 과정

WordCloud는 내부적으로 다음과 같은 순서로 동작한다.

```
텍스트 입력
        ↓
토큰화(Tokenization)
        ↓
불용어 제거
        ↓
연어(Collocation) 탐지
        ↓
단어 빈도 계산
        ↓
상위 단어 선택
        ↓
단어 크기 계산
        ↓
큰 단어부터 배치
(적분 이미지 + 나선형 탐색)
        ↓
색상 지정
        ↓
RGB 이미지 렌더링
        ↓
plt.imshow()로 출력
```

즉, 워드클라우드는 단순히 단어를 크게 출력하는 것이 아니라 **텍스트 분석 → 빈도 계산 → 크기 결정 → 위치 탐색 → 이미지 생성**의 과정을 거쳐 최종 시각화를 수행한다.
